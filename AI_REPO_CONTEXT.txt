AI REPO CONTEXT

Purpose
- Fast reference for AI agents working in this repository.

Project Summary
- Event-driven ecommerce backend.
- Main stack: TypeScript, AWS Lambda, SQS, EventBridge, DynamoDB, Terraform.
- Source code lives mainly in `src/`.
- Infra code lives in `terraform/`.

Current High-Level Structure
- `src/domain/`
  - Core business objects and domain errors.
- `src/application/`
  - Use cases, DTOs, repository interfaces, shared app utilities, event definitions.
- `src/infrastructure/`
  - AWS/DynamoDB implementations.
- `src/presentation/`
  - Lambda handlers for API Gateway and SQS.
- `skills/`
  - Local Codex skills documenting repo-specific workflows and patterns.

Important Domain/Application Conventions
- Domain file path pattern:
  - `src/domain/<Domain>/<Domain>.ts`
- Application file path pattern:
  - `src/application/<Domain>/`
- Repository interfaces:
  - `src/application/<Domain>/repositories/I<Domain>Repository.ts`
- Repository barrel:
  - `src/application/<Domain>/repositories/index.ts`
- Use case folder name is exactly:
  - `Usecase`

DTO Rules
- DTOs do NOT belong in `src/domain`.
- Repository-local DTOs should usually live in:
  - `src/application/<Domain>/repositories/I<Domain>Repository.ts`
- Broader application/event/input-output DTOs may live in:
  - `src/application/<Domain>/dtos/*.ts`
- DTO property names must be PascalCase.
- Examples:
  - `Id`
  - `UserId`
  - `ProductId`
  - `CreatedAt`
  - `UpdatedAt`
  - `SentAt`

Domain Code Rules
- Domain internal properties use camelCase.
- Domain state should be private by default.
- Prefer ECMAScript private fields such as `#status`, `#updatedAt`, `#id`.
- If outside code needs to read a domain value, expose a `get` instead of making the field public.
- Do NOT use underscore-prefixed properties in domain classes.
- Avoid patterns like:
  - `_status`
  - `_updatedAt`
- Prefer simple names like:
  - `status`
  - `updatedAt`
- Avoid redundant names like:
  - `currentStatus`
  - `statusValue`
  - `updatedAtValue`

Domain Type Alias Rules
- Do not leave loose top-level aliases such as:
  - `type OrderId = string`
  - `type UserId = string`
- Put domain-local aliases inside:
  - `export namespace <Domain>Domain`
- Example:
  - `OrderDomain`
  - `InventoryDomain`
  - `NotificationDomain`

Mapping Rules
- Domain entities may use camelCase properties internally.
- `toDto()` is responsible for mapping domain names to PascalCase DTO keys.
- Keep mapping explicit and predictable.

Important Architectural Notes
- Domain contains business rules and state transitions.
- Application use cases orchestrate flows.
- Repository interfaces belong to application layer.
- Infrastructure implements repository contracts.
- Presentation handlers parse input, instantiate dependencies, call use cases, log, and handle expected errors.

SQS/Event Notes
- Existing SQS listener pattern is documented in:
  - `skills/create-sqs-listener/SKILL.md`
- Event and queue naming should stay aligned across:
  - `src/application/Event/Events.ts`
  - handlers in `src/presentation/aws-sqs/`
  - Terraform resources

Terraform Attention Points
- DynamoDB table/index names must match repository assumptions.
- Follow existing Terraform style for DynamoDB GSIs.
- Before changing persistence contracts, inspect:
  - `terraform/aws_dynamodb.tf`
  - related repository implementation
  - related DTO contract

Before Editing a Module
- Check whether a DTO already exists and where it belongs.
- Check whether the module uses repository-local DTO or shared DTO file.
- Check whether the domain already has a namespace for local types.
- Check whether any property name violates the no-underscore rule.
- Check whether `toDto()` returns PascalCase keys.
- Check whether the repository `index.ts` export matches current project style.

When Creating a New Domain Module
- Create domain class in `src/domain/<Domain>/<Domain>.ts`
- Create `export namespace <Domain>Domain` for local type aliases.
- Create repository interface in `src/application/<Domain>/repositories/I<Domain>Repository.ts`
- Create `index.ts` in repository folder.
- Add `Usecase/` only if there is application orchestration to implement.
- Place DTO in repository file unless it is a broader cross-layer contract.

Definition Of Done For Most Changes

- DTO keys are PascalCase.
- Domain has no underscore-prefixed properties.
- Domain-local type aliases are inside `<Domain>Domain` namespace.
